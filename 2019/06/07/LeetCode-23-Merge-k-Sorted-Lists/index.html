LeetCode 23. Merge k Sorted Lists 2019-06-07

题目描述

合并 k 个排序链表，返回合并后的排序链表。请分析和描述算法的复杂度。

tag

合并题 链表 分而治之 堆

样例

算法1

(分而治之 自底向上) O(Nlogk)。
思路

将 k 个链表配对并将同一对中的链表合并。
每一轮合并以后，链表数少一半，重复这一过程，
重复这一过程 log2K 次。
直到我们得到了最终的有序链表。

复杂度分析

O(n) 的时间内合并两个有序链表，重复这一过程 log2K 次。所以最终O(Nlogk)。O(1) 的空间实现两个有序链表的合并。

算法3

(堆) O(Nlogk)
思路

先用每个链表的第一个节点的val和其下标初始化一个小顶堆，将堆顶插入结果链表末尾，同时将其原来所在链表的下一个节点插入堆中。

复杂度分析

弹出操作时，比较操作的代价会被优化到 (logk) 。同时，找到最小值节点的时间开销仅仅为 O(1)。最后的链表中总共有 N 个节点，堆需要 O(k) 的空间。
