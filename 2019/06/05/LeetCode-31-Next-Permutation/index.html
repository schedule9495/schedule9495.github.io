LeetCode 31. Next Permutation 2019-06-05

题目描述

实现获取下一个排列的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列。

如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。

样例

算法

O(n)

先从后往前，找到第一个发生逆序的位置k（i - 1）和i，nums[k] < nums[i]。如果i = 0，说明不存在下一个排序，此时是一个降序排列，如[4，3，2，1],返回翻转的结果[1，2，3，4]。
然后从后往前，找到第一个大于nums[k]的元素下标j，交换nums[k]和nums[j]，为什么交换呢，nums[j]原来比nums[k]要大，将一个较大的数跟前面的一个较小的数交换，使得这个数整体变大，这也是得到下一个排列的要求，但下一个排序仅仅是比当前数只大一点的数，不需要现在这么大，所以还需要最后一步让它再变小一点点，即让[k + 1:]位置上的序列反转，因为原来按照上面第一步第二步处理过后，[k + 1:]位置上的数现在应该是降序的，例如[3,2,1]，现在反转它得到[1，2，3]，使得这个数整体变小一点点，恰好得到下一个排列

LintCode 51. 上一个排列

算法

O(n)

第一步，从后往前找，找到第一个k（i - 1）和i，使得nums[k] > nums[i]
第二步，从后往前找，找到第一个j，使得nums[j] < nums[k], 交换j，k位置上的元素
第三步，将[k + 1:]位置上的元素反转

只要在上题代码上改两个大于小于号即可，不过LintCode要返回nums，而Leetcode原地修改nums不返回需要注意一下。
